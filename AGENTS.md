# AGENTS.md

Instructions for Codex and other coding agents working in this repository.

## Project Overview

This repository is a fork of veRL v0.4.1, an LLM reinforcement learning training framework based on Ray, PyTorch FSDP/Megatron-LM, and vLLM/SGLang.

## Repository Boundaries

- `verl/` contains upstream implementation. Avoid modifying files under `verl/` unless the task explicitly requires a framework-level change.
- `recipe/` contains custom experiments and recipes. Prefer adding or changing custom work here.
- `examples/` contains upstream launch scripts and experiment configs.
- Before modifying an existing recipe, read that recipe's README or nearby documentation.
- For new experiments, create one directory per experiment under `recipe/<experiment_name>/`.

## Important Local Paths

- Models: `/data3/yyy/models/`
- Data: `/data3/yyy/verl/data/`
- Main custom recipe: `recipe/RLSD/`
- Other custom recipes include `recipe/train_gsm8k/`, `recipe/math_challenger_solver/`, and `recipe/math_challenger_solver_judge/`.

## Upstream Architecture Pointers

- `verl/single_controller/`: Ray-based distributed orchestration, including `WorkerGroup`, `ResourcePool`, and dispatch/execute decorators.
- `verl/protocol.py`: `DataProto`, the shared transport format built around TensorDict, non-tensor dicts, and `meta_info`.
- `verl/workers/actor/dp_actor.py`: PPO actor update logic, loss computation, gradient accumulation, and micro-batching.
- `verl/workers/rollout/`: rollout generation through vLLM or SGLang.
- `verl/workers/critic/`: critic training.
- `verl/workers/reward_manager/`: reward computation through the registry mechanism.
- `verl/trainer/ppo/ray_trainer.py`: `RayPPOTrainer`, the main RL training loop.
- `verl/trainer/ppo/core_algos.py`: advantage estimation and policy loss implementations.
- `verl/trainer/config/`: Hydra/OmegaConf training configs.
- `verl/models/`: model registration for transformers and Megatron.

## Algorithms

The repository supports PPO, GRPO, DAPO, RLOO, REINFORCE++, ReMax, PRIME, PF-PPO, VAPO, DrGRPO, KL_Cov, and Clip_Cov.

## EMNLP 2026 Paper

- Paper path: `papers/emnlp2026/main.tex`.
- Current target: EMNLP 2026, deadline May 25 UTC-12.
- Main claim: COT-reference self-distillation can degrade mathematical reasoning by suppressing epistemic verbalization; masking epistemic token positions in the per-token KL loss mitigates this degradation.
- Current paper structure, result tables, figure list, and compile SOP are documented in `recipe/RLSD/CONTEXT.md` Section 13.
- Figures are generated by `recipe/RLSD/figures/generate_figures.py` and copied to `papers/emnlp2026/figures/`.
- AIME evaluation in the current code/paper is `acc@12`: macro-average fraction of correct samples per problem over 12 trials, not pass@12.
- ACL style provides the bibliography style; do not add a duplicate `\bibliographystyle{...}` in `main.tex`.
- When editing the paper, keep claims defensible: prefer "mitigates" over "prevents", avoid "necessary and sufficient", and align formulas with the implemented teacher-to-student clipped full-distribution KL.
- Compile in a clean temporary copy if project-directory TeX writes fail, then copy `main.pdf` back to `papers/emnlp2026/main.pdf`.
