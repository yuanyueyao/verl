\title{Epistemic Tokens as Functional Anchors: Why Self-Distillation Fails in Mathematical Reasoning and How to Fix It}

% Anonymous submission — authors hidden
\author{}

\begin{document}
\maketitle

\begin{abstract}
On-policy self-distillation is becoming a promising post-training paradigm for large language models.
In this paradigm, the model first samples its own response as a student, and then conditioned on additional context, provides token-level supervision on that response as a teacher.
However, we find it can severely degrade mathematical reasoning.
We trace this degradation to a specific mechanism: the teacher, conditioned on a reference solution, systematically suppresses student's \textit{epistemic tokens} such as ``Wait'', which express uncertainty and support self-verification during complex reasoning.
Although these tokens occupy only a small fraction of model outputs, they receive a severe suppression signal from the teacher, pushing the student toward overconfident, less reflective responses.
We further show that suppression severity depends on how detailed the reference solution is.
Based on above analysis, we propose \textit{masked self-distillation}, a minimal modification that excludes epistemic tokens from the distillation loss.
This simple intervention preserves reflective reasoning behavior.
Experiments across AIME24, AIME25, and MATH-500 demonstrate the superiority of masked self-distillation over naive self-distillation.
\end{abstract}

\section{Introduction}
% ═══════════════════════════════════════════════════════════════

On-policy self-distillation has recently emerged as a post-training paradigm for LLMs \cite{hubotter2026reinforcement,zhang2026opsdl,jin2026unisd,zhang2026simple}.
In this paradigm, the same model acts as both student and teacher under asymmetric conditioning: the teacher receives ground-truth solutions or environment feedback as privileged information, while the student sees only the problem.
The student first samples its own response. The teacher, now seeing the reference, computes a next-token distribution at each position of that response. The student is then trained to match this teacher distribution via per-token KL divergence.\footnote{See Equation~\ref{eq:sd_loss} for the formal definition.}

\begin{figure}[t]
\centering
\includegraphics[width=\columnwidth]{figures/teaser.pdf}
\caption{Failure mode of naive self-distillation. Distillation signal suppresses epistemic tokens, pushing the student model toward overconfident, less reflective reasoning.}
\label{fig:teaser}
\end{figure}

Recent work has demonstrated strong promising results with this paradigm.
SDPO~\cite{hubotter2026reinforcement} shows that dense token-level self-distillation outperforms Reinforcement Learning with Verifiable Rewards (RLVR)~\cite{guo2025deepseek,yu2025dapo,shao2024deepseekmath} and Supervised Fine-Tuning (SFT)~\cite{ouyang2022training,wei2022flan,chung2024scaling} baselines across science~QA, tool-use, and code generation benchmarks, while substantially shortening model outputs.
SimpleSD~\cite{zhang2026simple} reports similar gains and length reduction for code generation, OPSDL~\cite{zhang2026opsdl} extends self-distillation to long-context learning, and UniSD~\cite{jin2026unisd} demonstrates broad applicability across diverse adaptation benchmarks.
These successes suggest that self-distillation can be effective in settings where concise, confident outputs are sufficient.

However, this paradigm harbors a striking and structured failure mode.
When applied to challenging mathematical reasoning benchmarks~\cite{hendrycks2021measuring,cobbe2021training,patel2021svamp,saxton2019analysing}, self-distillation can severely degrade out-of-distribution performance.
This contrast---self-distillation appears effective in tool, chemistry, world knowledge, or code-like settings yet fails in our mathematical reasoning setting---raises a fundamental question:
\textit{Why does self-distillation degrade mathematical reasoning?}

We trace the root cause to the suppression of epistemic tokens---the model's explicit uncertainty expressions like ``Wait,'' ``Hmm,'' ``perhaps,'' and ``maybe.''
When the teacher sees the correct solution, it produces confident reasoning with minimal uncertainty. The student imitates this style and loses the exploratory behavior needed to recover from errors.
A token-level analysis reveals why: the teacher systematically assigns lower probability to student-generated epistemic tokens, creating a suppression signal that pushes the student toward less reflective continuations.
Figure~\ref{fig:teaser} illustrates this contrast.This mechanism grows more severe when teacher is provided with richer reference context: concise solutions cause mild suppression, while solutions with full reasoning traces cause extreme suppression.

This diagnosis directly motivates a simple intervention: \textit{epistemic token masking}. During self-distillation, we exclude positions where the student generates an epistemic token from the distillation loss, removing the suppressive signal while preserving ordinary distillation elsewhere.

Our contributions are:
\begin{enumerate}[leftmargin=*,nosep]
    \item \textbf{Functional-anchor diagnosis}: We identify epistemic tokens as sparse but important anchors for mathematical self-verification and show that self-distillation severely concentrates on these epistemic tokens, thereby degrading reasoning.

    \item \textbf{Reference-context analysis}: We show that epistemic-token suppression is controlled by the teacher's privileged context: concise solutions induce mild suppression, whereas full reasoning traces cause severe suppression.

    \item \textbf{Masked self-distillation}: We introduce a drop-in loss modification that masks epistemic-token positions during self-distillation. The resulting objective protects self-verification behavior.
\end{enumerate}
\begin{figure*}[t!]
\centering
\includegraphics[width=\textwidth]{figures/emnlp_overview.pdf}
\caption{\textbf{Overview of epistemic token masking.}
Using a simple math problem as a running example, the figure contrasts naive self-distillation (top) with our masked self-distillation method (bottom).

\section{Background and Related Work}
\label{sec:background}
% ═══════════════════════════════════════════════════════════════

\subsection{Self-Distillation for LLM Post-Training}

Self-distillation~\cite{snell2022learning} distills privileged context into model parameters by minimizing a teacher-to-student KL divergence.
In the on-policy setting~\cite{agarwal2024onpolicy}, the model serves as both teacher and student under asymmetric conditioning.
Let the student condition only on the problem $x$: $\pi_\theta^{\text{S}}(\cdot \mid x, \hat{y}_{<t}) = \pi_\theta(\cdot \mid x, \hat{y}_{<t})$.
The teacher additionally receives privileged information $s$ (e.g., a reference solution):
$\pi_\theta^{\text{T}}(\cdot \mid x, s, \hat{y}_{<t}) = \pi_\theta(\cdot \mid x, s, \hat{y}_{<t})$.
Training minimizes per-token KL with the teacher stopped-gradient:

{\small
\begin{equation}
    \mathcal{L}_{\text{SD}}(\theta) = \sum_{t=1}^{L} \text{KL}\bigl(
        \text{sg}\bigl(\pi_\theta^{\text{T}}(\cdot \mid x, s, \hat{y}_{<t})\bigr)
        \;\|\;
        \pi_\theta^{\text{S}}(\cdot \mid x, \hat{y}_{<t})
    \bigr)
\label{eq:sd_loss}
\end{equation}}

Recent work has produced a family of on-policy self-distillation variants.
SDPO~\cite{hubotter2026reinforcement} conditions the teacher on previously-generated correct trajectories or environment feedback, demonstrating that dense token-level KL can simultaneously improve accuracy and shorten responses.
UniSD~\cite{jin2026unisd}, OPSDL~\cite{zhang2026opsdl}, and SimpleSD~\cite{zhang2026simple} extend the paradigm to unified multi-task, long-context, and code-generation settings, while Ye et al.~\cite{ye2026onpolicy} distill external context into the student.
Further extensions address continual learning~\cite{shenfeld2026self}, memory-augmented agents~\cite{liu2026exploratory}, text feedback~\cite{song2026expanding}, and general privileged information~\cite{penaloza2026privileged}.

A common observation across these methods is that self-distillation often shortens model outputs.
This connects to a growing line of work on reasoning compression, which aims to reduce chain-of-thought length while preserving accuracy~{\cite{shrivastava2025sample,sang2026onpolicy,deng2026conpress,yang2026accordion,luo2026compress}}.
This paper asks a different question: when dense distillation pushes the student toward concise traces, which reasoning behaviors are lost?
\subsection{Epistemic Tokens and Related Directions}

Chain-of-thought prompting and zero-shot reasoning elicit extended intermediate reasoning traces in large language models~\cite{wei2022chain,kojima2022large}, and later prompting methods further improve reasoning through diverse sampled chains, problem decomposition, and tree-structured exploration~\cite{wang2023selfconsistency,zhou2023leasttomost,yao2023tree}.
We use \textit{epistemic tokens} to denote surface forms such as ``wait,'' ``hmm,'' and ``perhaps'' that externalize uncertainty, backtracking, and self-checking during such reasoning.
We treat them as functional anchors for self-verification: signals that often appear when the model checks, revises, or recovers its reasoning.

Prior work on verbalized uncertainty shows that language models can express uncertainty in natural language and that such expressions can be studied through calibration, verbalized probabilities, and factuality-oriented fine-tuning~\cite{lin2022teaching,kadavath2022language,tian2023fine}.
This supports treating uncertainty expressions as meaningful behavioral signals rather than mere style words.

Related work on self-correction studies whether models can detect and repair their own errors, with mixed results~\cite{madaan2023self,huang2023large}, while process reward models add step-level supervision for verification~\cite{lightman2023lets,uesato2022solving,cobbe2021training}.
These directions motivate our focus on local checking and revision behavior inside reasoning traces.
We ask whether dense self-distillation preserves or suppresses the epistemic tokens associated with that behavior.

\section{Diagnosis: Why Self-Distillation Degrades Reasoning}
\label{sec:diagnosis}
% ═══════════════════════════════════════════════════════════════

Before proposing a solution, we first ask what self-distillation is optimizing away.
Our diagnosis starts from a simple view: not all response tokens play the same role in reasoning.
Some tokens carry answers or local computations, while others mark uncertainty, checking, and revision.
The dense token-level objective treats these roles uniformly, which creates a specific failure mode at epistemic-token positions.

\subsection{Domain Contrast: Success and Failure of Self-Distillation}

Prior self-distillation work reports strong results in science~QA, tool-use, code generation, long-context learning, and broad adaptation benchmarks~\cite{hubotter2026reinforcement,zhang2026opsdl,jin2026unisd,zhang2026simple}.
In these settings, concise and confident responses are often enough once the environment feedback, relevant fact or reference solution is available.
Mathematical reasoning is different: solving hard problems often requires checking intermediate steps, revising false starts, and trying alternative paths.
Epistemic tokens are part of this behavior rather than just style, so suppressing them can remove useful support for complex reasoning.

\subsection{Token-Level Signal Concentration}

To quantify the teacher's suppression of epistemic tokens, we conduct a controlled forward-pass probe.
Using DeepSeek-R1-Distill-Qwen-7B~\cite{guo2025deepseek}, we sample student-generated responses on AIME-style problems, then compute both the teacher (conditioned on a full reasoning trace) and student (conditioned only on the problem) next-token probability distributions at every token position.

Table~\ref{tab:prob_probe} reports the mean probability assigned to the sampled student-generated token at each position, grouped by whether that token is epistemic or non-epistemic.

\bottomrule

The teacher assigns only \textbf{0.127} mean probability to epistemic tokens---5.9$\times$ lower than the 0.751 assigned to non-epistemic tokens.
The student, in contrast, maintains a higher epistemic probability of 0.417.
This creates a concentrated token-level self-distillation signal at epistemic-token positions, where the student response contains uncertainty or reflection tokens but the teacher's full next-token distribution shifts probability mass toward non-epistemic continuations.
In other words, this signal pushes the student away from reflective continuations and toward the teacher's more confident continuations.

Figure~\ref{fig:token_kl_strip} provides a qualitative illustration on a representative response.
The top heatmap colors each sampled student token by its token-level self-distillation signal under naive self-distillation, with outlined epistemic tokens and error/correction positions.
Error positions and epistemic-token positions both receive high signal under naive self-distillation, while many normal positions receive lower signal.
The bottom bar chart shows that their mean signal values are comparable (0.147 vs.\ 0.123), suggesting that the dense token-level objective does not tell useful correction apart from suppression of self-verification.

Epistemic tokens occupy roughly 12\% of response tokens but receive $\sim$52\% of the aggregate token-level self-distillation signal---a 4.3$\times$ concentration effect.
Operationally, this signal is the per-position teacher-to-student next-token KL term in the self-distillation objective.
Thus, the objective does not treat all tokens equally; it concentrates on epistemic tokens that implement hesitation, checking, and recovery.

What does the teacher prefer instead of epistemic tokens?
Qualitatively, it tends to replace reflective continuations with more direct ones.
For example, a student trace may begin with ``Hmm'' or recover from an error with ``Wait, let me check,'' while the teacher conditioned on the reference solution proceeds directly with ``Since'' or ``Therefore.''
This shows that the signal can redirect the student away from checking and revision, not only away from wrong computations.

\subsection{Reference Context Matters}

Finally, we investigate how the \textit{type} of reference solution affects epistemic suppression.
We compare four reference conditions:

\begin{itemize}[leftmargin=*,nosep]
    \item \textbf{None(Student)}: Plain problem, no privileged information.
    \item \textbf{GT only}: Problem + correct answer only.
    \item \textbf{Clean solution}: Problem + concise step solution.
    \item \textbf{Full reasoning trace}: Problem + complete reasoning trace.
\end{itemize}

Table~\ref{tab:ref_context} reports response length, epistemic-token frequency, and conditional answer accuracy under each reference condition.

\begin{table*}[t!]

First, \textbf{clean reference solutions produce mild epistemic suppression}, reducing epistemic tokens from 728 to 459 and their response percentage from 14.0\% to 10.0\%.
Second, \textbf{full reasoning traces cause extreme suppression}, reducing epistemic tokens to 146 and this percentage to 5.0\%, while simultaneously inflating accuracy to 90\%.

In short, epistemic tokens are infrequent yet signal-heavy and functionally important. This diagnosis directly motivates our intervention.

\section{Method: Epistemic Token Masking}
% ═══════════════════════════════════════════════════════════════

The diagnosis above reveals the root cause: the token-level self-distillation signal concentrates on epistemic tokens.
The method follows directly from this insight.
we mask self-distillation at positions where the sampled student token belongs to epistemic token.

\subsection{Epistemic Token Set}
\label{sec:epistemic_token_set}

We define a lightweight set of epistemic tokens as an operational approximation:
\begin{equation}
\begin{aligned}
    \mathcal{E} = \{&\text{wait}, \text{hmm}, \text{perhaps}, \text{maybe}, \text{actually}, \\
                    &\text{alternatively}, \text{seems}, \text{might}, \text{likely}, \text{check}\}
\end{aligned}
\end{equation}

The set is intentionally small and interpretable: it covers common uncertainty, reconsideration, and self-checking tokens highlighted by the diagnostic probes.
To handle surface-form variation (capitalization, punctuation, surrounding whitespace), we enumerate common variants of each token and encode them with the model's tokenizer.
This yields a set of token IDs $\mathcal{I}$, cached and reused across training steps.

\subsection{Masked Self-Distillation Loss}

Let $\hat{y} = (\hat{y}_1, \dots, \hat{y}_L)$ be the student's generated response of length $L$, and let $\pi_\theta^{\text{S}}$, $\pi_\theta^{\text{T}}$ denote the student and teacher distributions as defined in~Section~\ref{sec:background}.
We construct a binary token mask $M \in \{0,1\}^L$:
\begin{equation}
    M_t = \begin{cases}
        0 & \text{if } \hat{y}_t \in \mathcal{I} \\
        1 & \text{otherwise}
    \end{cases}
\end{equation}

In implementation, we use the same clipped full-distribution teacher-to-student KL for both naive self-distillation and masked self-distillation; masking is the only difference between the two variants.
Let $p^{\text{T}}_{t,v}$ and $p^{\text{S}}_{t,v}$ denote the teacher and student probabilities for vocabulary item $v$ at response position $t$.
The per-token self-distillation signal is:
\begin{equation}
    \ell_t =
    \sum_{v \in \mathcal{V}}
    \min\left(
        p^{\text{T}}_{t,v}
        \log \frac{p^{\text{T}}_{t,v}}{p^{\text{S}}_{t,v}},
        \tau
    \right),
\label{eq:clipped_kl}
\end{equation}
where $\tau$ is the per-vocabulary-item clipping threshold in our experiments.
This clipping prevents a small number of sharp vocabulary items from dominating the full-distribution signal.
The masked self-distillation loss sums this signal over non-masked positions:
\begin{equation}
\begin{aligned}
    \mathcal{L}_{\text{masked}}(\theta)
    &=
    \frac{\sum_{t=1}^L M_t \cdot \ell_t}
    {\max(\sum_{t=1}^L M_t,\; \varepsilon)} ,
\end{aligned}
\label{eq:masked_loss}
\end{equation}

where $\varepsilon = 10^{-8}$ guards against the degenerate case where all tokens are epistemic.
The denominator normalizes by the number of \textit{non-masked} tokens, ensuring that masking does not artificially reduce the loss magnitude.
For memory efficiency, we compute $\ell_t$ in response-position chunks rather than materializing the full $(L \times |\mathcal{V}|)$ distribution tensor at once. The mask is applied to each chunk before accumulation.
Algorithm~\ref{alg:masked_opsd} summarizes the resulting training loop.


\begin{algorithm}[t]
\caption{Masked Self-Distillation}
\label{alg:masked_opsd}
\small
\begin{algorithmic}[1]
\setlength{\itemsep}{2pt}
\Require Training set $\mathcal{S}=\{(x_i,s_i)\}$; model $\pi_\theta$; epistemic token-ID set $\mathcal{I}$; clip threshold $\tau$; stability constant $\varepsilon$.
\State Let $\pi_\theta^{S}(\cdot \mid x)$ and $\pi_\theta^{T}(\cdot \mid x, s)$ denote the same model $\pi_\theta$ under different conditioning.
\While{not converged}
    \State Sample minibatch $\mathcal{B} \subset \mathcal{S}$
    \ForAll{$(x, s) \in \mathcal{B}$}
        \Statex \textbf{\textit{Rollout and mask.}}
        \State Sample student rollout $\hat{y} \sim \pi_\theta^{S}(\cdot \mid x)$
        \State \textcolor{red}{$M_t \gets \mathbf{1}[\hat{y}_t \notin \mathcal{I}]$, for $t = 1, \dots, |\hat{y}|$}
               \Comment{\textcolor{red}{mask}}
        \Statex \textbf{\textit{Token-level self-distillation signal.}}
        \For{$t = 1, \dots, |\hat{y}|$}
            \State $p^{T}_t \gets \mathrm{sg}\!\left[\pi_\theta^{T}(\cdot \mid x, s, \hat{y}_{<t})\right]$
            \State $p^{S}_t \gets \pi_\theta^{S}(\cdot \mid x, \hat{y}_{<t})$
            \State $\ell_t \gets \displaystyle\sum_{v \in \mathcal{V}}
                   \min\!\left(p^{T}_{t,v}\log\tfrac{p^{T}_{t,v}}{p^{S}_{t,v}},\; \tau\right)$
        \EndFor
        \Statex \textbf{\textit{Masked objective.}}
        \State \textcolor{red}{$\displaystyle
                   \mathcal{L}(x;\theta) \gets
                   \frac{\sum_{t=1}^{|\hat{y}|} M_t\, \ell_t}
                        {\max\!\left(\sum_{t=1}^{|\hat{y}|} M_t,\; \varepsilon\right)}$}
    \EndFor
    \Statex \textbf{\textit{Update.}}
    \State $\theta \gets \theta - \eta\, \nabla_\theta\,
           \dfrac{1}{|\mathcal{B}|}\!\sum_{(x,s) \in \mathcal{B}} \mathcal{L}(x;\theta)$
\EndWhile
\end{algorithmic}
\end{algorithm}

\section{Experiments}
% ═══════════════════════════════════════════════════════════════

We evaluate masked self-distillation across two model scales under full-reasoning-trace self-distillation to test both the generality of the degradation phenomenon and the effectiveness of our intervention.

\subsection{Setup}

\textbf{Models and data.}
We use high reasoning ability models DeepSeek-R1-Distill-Qwen~\cite{guo2025deepseek} at 1.5B and 7B scales.
Training data are drawn from the mathematical reasoning portion of OpenThoughts \cite{guha2025openthoughts}, with up to 30K problems and solution traces.
The teacher receives the available full reasoning trace as privileged context, while other types of reference contexts (e.g., concise solutions, correct answer only) are used only in the diagnostic probe.

\textbf{Baselines and evaluation.}
We compare masked self-distillation against the base model and naive self-distillation, which uses the same teacher-to-student KL objective without masking.
Following self-distillation with privileged feedback or context \cite{hubotter2026reinforcement}, each step samples 12 problems and one student rollout per problem.
We evaluate on AIME 2024, AIME 2025~\cite{guo2025deepseek}, and MATH-500~\cite{hendrycks2021measuring}.
For AIME, we report \textit{avg@12}: average correctness over 12 sampled responses per problem, macro-averaged over problems.
For MATH-500, we report pass@1.
Answers are extracted from \texttt{\textbackslash boxed\{\}} and verified against ground truth using Math-Verify~\cite{mathverify2024}.

\textbf{Implementation details.}
Our implementation is based on the veRL codebase~\cite{sheng2024hybridflow}.
All self-distillation runs train for 100 steps on 8 NVIDIA A800-80GB GPUs.
We conduct multiple runs and report averaged results.
Additional details are reported in Appendix~\ref{app:hyperparams}.

\subsection{Main Results}

Table~\ref{tab:cot_results} reports the main results, testing whether masking mitigates the degradation predicted by our full-reasoning-trace diagnosis.

\subsection{Analysis}

\paragraph{Epistemic token retention.}
We track the frequency of epistemic tokens in student responses throughout training.
Figure~\ref{fig:dynamics} (left) shows that under naive self-distillation, epistemic tokens collapse early and only partially recover later.
This supports a two-stage dynamic: an early concentrated token-level self-distillation signal at epistemic-token positions suppresses reflection and self-checking, while the broader signal over non-epistemic positions later restores some task-relevant behavior.
Under masked self-distillation, epistemic-token usage is largely preserved, keeping the student model more reflective.

\paragraph{Response length dynamics.}
Naive self-distillation with full reasoning traces also causes response length to fall sharply early in training, followed by only partial recovery (Figure~\ref{fig:dynamics}, right).
In contrast, masked self-distillation keeps longer reasoning traces throughout training.
Together with the accuracy and epistemic-token trends, the early length drop under naive self-distillation suggests a loss of self-checking rather than useful compression: the model writes less, expresses less uncertainty, and achieves lower accuracy.
By preserving epistemic tokens, masked self-distillation keeps both accuracy and longer reasoning traces without an explicit length objective.

\section{Discussion}
% ═══════════════════════════════════════════════════════════════

\paragraph{Why does such a simple intervention work?}
Our diagnostic findings provide a clear causal account: self-distillation degradation is driven by a concentrated mechanism—the teacher's systematic suppression of a small, specific set of tokens.
These epistemic tokens constitute roughly 12\% of response tokens in our math-reasoning probe but account for a disproportionate percentage of the aggregate token-level self-distillation signal.
By protecting only these positions, we eliminate the source of degradation without meaningfully reducing the token-level self-distillation signal elsewhere valuable.

\paragraph{Domain generality.}
Our findings explain why self-distillation can behave differently in tool-use, code generation, and mathematical reasoning.
Tool and code tasks often provide executable feedback, syntactic targets, or API-grounded response behavior, so a concise teacher trace can be helpful rather than harmful.
In mathematical reasoning, however, exploration, self-verification, and recovery from intermediate errors depend on explicit uncertainty expression. Suppressing epistemic tokens can therefore be highly damaging.
Because epistemic tokens are rare in code-like domains, we expect the mask to have little effect there; verifying this empirically is an important direction for future work.

\paragraph{Reference context as a control knob.}
Our diagnostic probe (Section~\ref{sec:diagnosis}) suggests that the type of privileged information provided to the teacher modulates epistemic suppression severity.
Concise solutions produce mild suppression (37\%); full reasoning traces cause extreme suppression (80\%).
This suggests that practitioners can choose reference contexts to balance learning speed and reasoning preservation, and that masking becomes increasingly important as the reference context becomes richer.

\paragraph{Limitations.}
Our study is limited to the DeepSeek-R1-Distill-Qwen reasoning model family.
The predefined epistemic token set may also miss novel uncertainty expressions, though our experiments suggest the current set captures the most impactful tokens.
Adaptive token identification, per-domain epistemic token sets or entropy-based masking are promising directions for future work.

\paragraph{Broader implications.}
Our findings point to a broader principle: dense token-level objectives are epistemically blind.
They apply the same formal signal to functionally distinct tokens, even when those tokens play different roles in the reasoning process.
This blind spot is not unique to self-distillation.
Similarly, the KL penalty that constrains the policy toward a reference model in RLHF~\cite{ziegler2019finetuning,stiennon2020learning,bai2022training,ouyang2022training}, and the cross-entropy loss toward a target distribution in SFT, deserves further consideration.
Our results suggest that post-training objectives should account for token function.


% ═══════════════════════════════════════════════════════════════
\section{Conclusion}
% ═══════════════════════════════════════════════════════════════
In this paper, we identify a failure mode of on-policy self-distillation in mathematical reasoning. 
We show that navie distillation degrades reflective reasoning ablitity through punishing epistemic tokens, thus suppressing the hesitation, revision, and self-verification behaviors needed for complex reasoning.
To address this issue, we propose masked self-distillation, a simple objective modification that removes the loss term at epistemic-token positions while preserving ordinary distillation elsewhere. 
Experiments on AIME24, AIME25, and MATH-500 show that masked self-distillation mitigates the degradation of naive self-distillation and preserves reflective reasoning behavior. 
These results suggest that post-training objectives may need to account for token function, rather than treating all tokens similarly.

You are a helpful assistant.

Problem: \textcolor{PromptVar}{\{question\}}\\[3pt]
Please reason step by step, and put your final answer within \textbackslash boxed\{\}.
\end{promptbox}

\begin{promptbox}{Teacher prompt}
Problem: \textcolor{PromptVar}{\{question\}}\\[3pt]
Below is a verified reference solution. Use it to reason about the problem. Your own response wording may differ.\\[3pt]
\textcolor{PromptVar}{\{reference\_solution\}}\\[3pt]
Please reason step by step, and put your final answer within \textbackslash boxed\{\}.
\end{promptbox}

Teacher and student share the same parameters but differ in conditioning: the teacher receives the reference solution as privileged context, while the student sees only the problem. The teacher does not generate tokens during training---its next-token distribution is computed implicitly through a single forward pass that conditions on the reference solution. The student then minimizes its KL divergence to this distribution at each position (except masked epistemic positions). The teacher conditions on the \texttt{COT\_Reason} field of OpenThoughts~\cite{guha2025openthoughts}.

\section{Case Study: Domain Error from Suppressed Verification}
\label{app:case}

Below we contrast responses from the base model (DeepSeek-R1-Distill-Qwen-1.5B, step~0), the naive self-distilled model, and the masked self-distilled model (each after 100 self-distillation steps) on a representative MATH problem. Outputs are lightly condensed for presentation; the full reasoning traces exhibit the same patterns at greater length.
Epistemic tokens (e.g., ``Hmm,'' ``Wait,'' ``check'') are \textcolor{orange}{highlighted in orange}; errors in \textcolor{red}{red}.

\begin{problemblock}{Problem (MATH-500)}
\textit{Let \(f(x) = (\arccos x)^2 + (\arcsin x)^2\). Find the range of \(f(x)\). All functions are in radians.}
\end{problemblock}

\begin{caseblock}{Base model: reflective reasoning with self-correction}{blue}
Let's solve this. I remember that \(\arccos x + \arcsin x = \pi/2\) for any \(x \in [-1,1]\).

Let \(\theta = \arccos x\). Then \(\arcsin x = \pi/2 - \theta\), so
\(f(x) = \theta^2 + (\pi/2 - \theta)^2 = 2\theta^2 - \pi\theta + (\pi/2)^2\).

Since \(\theta = \arccos x\) and \(x \in [-1,1]\), \(\theta\) ranges over \([0, \pi]\).
This is a quadratic opening upwards; the vertex at \(\theta = \pi/4\) gives the minimum.

\textcolor{orange}{Wait,} let me verify the identity: at \(x=0\), \(\arccos 0 = \pi/2\), \(\arcsin 0 = 0\), sum \(\pi/2\). At \(x=1\), \(\arccos 1 = 0\), \(\arcsin 1 = \pi/2\). \textcolor{orange}{OK, correct.}

Minimum at \(\theta = \pi/4\): \(f = 2(\pi^2/16) - \pi^2/4 + \pi^2/4 = \pi^2/8\).

Endpoints: \(\theta = 0\) (i.e., \(x=1\)): \(f = 0 + (\pi/2)^2 = \pi^2/4\).
\(\theta = \pi\) (i.e., \(x=-1\)): \(f = 2\pi^2 - \pi^2 + \pi^2/4 = \pi^2 + \pi^2/4 = \textcolor{red}{3\pi^2/4}\).

\textcolor{orange}{Wait,} that's not right --- \(\pi^2\) is \(4\pi^2/4\), so adding \(\pi^2/4\) gives \(5\pi^2/4\), not \(3\pi^2/4\). I mistakenly combined the numerators without a common denominator. Correcting: \(f(\pi) = 5\pi^2/4\).

\textcolor{orange}{Let me double-check} by computing directly: \(f(-1) = (\arccos(-1))^2 + (\arcsin(-1))^2 = \pi^2 + (-\pi/2)^2 = \pi^2 + \pi^2/4 = 5\pi^2/4\). Good, that matches.

\textcolor{orange}{Let me also} find critical points via derivative:
\(f'(x) = \frac{2}{\sqrt{1-x^2}}[\pi/2 - 2\arccos x]\).
Setting to zero gives \(\arccos x = \pi/4\), i.e., \(x = \sqrt{2}/2\)---the same vertex.

\textcolor{orange}{Hmm, maybe} I should verify with the Extreme Value Theorem: \(f\) is continuous on \([-1,1]\), so it attains max and min at critical points or endpoints. The only critical point is at \(\sqrt{2}/2\) (minimum), and the maximum at \(x=-1\) gives \(5\pi^2/4\).

Thus range: \(\boxed{\left[ \dfrac{\pi^2}{8}, \dfrac{5\pi^2}{4} \right]}\).
\end{caseblock}

\begin{caseblock}{Naive self-distillation: confident error without verification}{red}
I need the range of \(f(x) = (\arccos x)^2 + (\arcsin x)^2\).
\textcolor{orange}{Hmm,} let me think.

Identity: \(\arccos x + \arcsin x = \pi/2\). Let \(a = \arccos x\), \(b = \arcsin x\). Then \(b = \pi/2 - a\).

\(f(x) = a^2 + (\pi/2 - a)^2 = 2a^2 - \pi a + \pi^2/4\).

This is a quadratic in \(a\). Since \(a = \arccos x\) with \(x \in [-1,1]\),
\textcolor{red}{\(a \in [0, \pi/2]\)}.

Vertex at \(a = \pi/4\): \(f = \pi^2/8\) (minimum).
Endpoints: \(a=0\) gives \(\pi^2/4\); \textcolor{red}{\(a=\pi/2\)} also gives \(\pi^2/4\).
Thus range: \(\boxed{\left[ \dfrac{\pi^2}{8}, \dfrac{\pi^2}{4} \right]}\).
\end{caseblock}
\begin{casenote}
The naive-distilled model incorrectly restricts the domain of \(\arccos x\) to \([0, \pi/2]\) instead of \([0, \pi]\), then only checks two endpoints---both yielding the same value---and confidently concludes the maximum is \(\pi^2/4\). The correct maximum at \(x=-1\) (\(5\pi^2/4\)) is never examined. The single ``Hmm'' at the start does not trigger verification of the domain assumption. This pattern is consistent with the early epistemic suppression observed in naive self-distillation (Figures~4--5): the model becomes overconfident in its initial framing and skips the boundary checks that the base model performs naturally.
\end{casenote}

\begin{caseblock}{Masked self-distillation: preserved verification, self-corrects}{orange}
I need the range of \(f(x) = (\arccos x)^2 + (\arcsin x)^2\).
\textcolor{orange}{Hmm,} let me think.

First, I remember that for any \(x \in [-1,1]\), \(\arccos x + \arcsin x = \pi/2\). That seems important. Let \(a = \arccos x\) and \(b = \arcsin x\). Then \(a + b = \pi/2\), so \(b = \pi/2 - a\).

Substituting: \(f(x) = a^2 + (\pi/2 - a)^2 = a^2 + (\pi^2/4 - \pi a + a^2) = 2a^2 - \pi a + \pi^2/4\).

This is a quadratic in \(a\). Since \(a = \arccos x\) and \(x \in [-1,1]\), when \(x = 1\) we have \(a = 0\); when \(x = -1\) we have \(a = \pi\). So \textbf{\(a \in [0, \pi]\)}.

The coefficient of \(a^2\) is positive, so the parabola opens upward---minimum at the vertex, maximum at an endpoint. Vertex: \(a = \pi/4\), which is inside \([0, \pi]\).

Now evaluate at the three candidate points:
\begin{itemize}
\item \(a = 0\): \(f = 0 - 0 + \pi^2/4 = \pi^2/4\).
\item \(a = \pi/4\): \(f = 2(\pi^2/16) - \pi^2/4 + \pi^2/4 = \pi^2/8\).
\item \(a = \pi\): \(f = 2\pi^2 - \pi^2 = \pi^2\), so \(f = \textcolor{red}{\pi^2/4}\).
\end{itemize}

\textcolor{orange}{Wait, hold on ---} I dropped the \(+\pi^2/4\) term! \(f(\pi) = 2\pi^2 - \pi^2 + \pi^2/4 = \pi^2 + \pi^2/4 = 5\pi^2/4\), not \(\pi^2/4\). That was a careless omission.

\textcolor{orange}{Let me verify} with the original function directly: at \(x = 1\), \(f = 0 + (\pi/2)^2 = \pi^2/4\). At \(x = -1\), \(f = \pi^2 + (-\pi/2)^2 = \pi^2 + \pi^2/4 = 5\pi^2/4\). \textcolor{orange}{All consistent.}

I considered the identity, transformed the function, checked the vertex and all three boundary candidates, caught a slip, and verified against the original. I'm confident the range is \([ \pi^2/8, 5\pi^2/4 ]\).

Thus: \(\boxed{\left[ \dfrac{\pi^2}{8}, \dfrac{5\pi^2}{4} \right]}\).
\end{caseblock}
\begin{casenote}
The masked-distilled model preserves the correct domain \([0, \pi]\) and the verification routine (``Wait, hold on,'' ``Let me verify,'' ``All consistent''). It checks all three candidate points rather than prematurely concluding from two coinciding endpoint values. The epistemic verification habit---suppressed under naive self-distillation---is retained because the per-token distillation signal at epistemic positions is excluded from the loss.
\end{casenote}
